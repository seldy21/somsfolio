HTML

  • w3c validator 검사
  • 주 브라우저는 chrome을 기준으로 작업
  • 자식, 자손 요소 들여쓰기. tab 한 번은 4번의 공백으로 설정
  • div 태그에는 필히 class 값 작성
  • class 값은 해당 콘텐츠와 최대한 연관된 단어로 작성. 비슷한 내용이 있는 경우 숫자로 구분
  • 비슷하게 연속되는 콘텐츠의 경우 ul 태그 내 li 태그로 작성
  • 최상위 공통 선택자 *는 되도록 사용하지 않음

css

  • 모든 페이지에 normalize.css와 fonts.css 연결
  • @charset “utf-8”; 선언 후 작성
  • 연결된 html 태그 작성 순서를 고려하여 순차대로 진행
  • width 등 사이즈에 되도록 소수점까지 표기하지는 않음
  • a 태그의 display 값을 block 또는 inline-block으로 설정하여 width 값, height 값 지정

script

  • jQuery 사용을 위해 jquery-3.6.0.min.js 파일 사용
  • 슬라이드 메뉴에 swiper-bundle.min.js 파일 사용
  • 제작한 js파일의 경우 head 영역에 작성, defer 값을 부여

files

  • 각 html과 연결하는 css, script는 외부문서로 연결
  • 파일명은 알파벳과 언더바로 구성
  • 같은 파일 종류대로 폴더에 정리

PROJECT GOAL

HLB 제약은 HLB KOREA 계열 제약 회사입니다. 일반 의약품, 전문 의약품을 포함하여 건강기능식품과 희귀질환 관련 치료제의 개발, 임상 실험을 진행하고 있습니다.

본 프로젝트에서는 제약회사가 주는 전문적인 이미지를 최대한 간결하게 나타내려 하였으며 화면은 크롬 브라우저를 기준으로 제작되었습니다.

  • 처음으로 보이는 메인 이미지에 vegas.js파일을 사용하여 동영상 효과 부여
  • 각 섹션에 해당하는 부분은 창 양 끝에 숨겨있다가 해당 스크롤에 도달하면 자연스럽게 등장. aos.js 파일 사용.
  • 섹션 제목 왼쪽에 붙는 긴 선은 css의 padding 값과 border 값으로 제어.
  • 대부분의 정렬에 flex 사용.
  • 웹 브라우저에 맞춰 제작하였으므로 body에는 min-width값으로 1200px 적용.

HBL 제약 웹 페이지 제작은 제가 처음으로 디자인과 퍼블리싱을 100% 자력으로 해낸 프로젝트입니다.

웹 화면을 기준으로 제작을 하다보니 높이의 값을 px로 주는 것이 좋을지, vh로 주는 것이 좋을지 고민을 많이 했습니다. 결론적으로는 px값으로 처리하였고, 나름대로 만족할만한 결과가 되었다고 생각합니다.

aos 스크립트를 적용할 때 New&Notice 부분이 나타나기 전에 오른쪽으로 스크롤이 생기는 문제가 있었습니다.이는 section의 부모요소인 main에 width 값 100%를 부여하고, overflow 를 hidden으로 처리하여 온전하게 만들었습니다.

모쪼록 해당 프로젝트가 보는 이로 편리함을 느끼게 하고, 디자인적으로 아름답게 보이기를 바랍니다.
