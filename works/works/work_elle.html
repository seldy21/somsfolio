HTML

  • w3c validator 검사
  • 주 브라우저는 chrome을 기준으로 작업
  • 자식, 자손 요소 들여쓰기. tab 한 번은 4번의 공백으로 설정
  • div 태그에는 필히 class 값 작성
  • class 값은 해당 콘텐츠와 최대한 연관된 단어로 작성. 비슷한 내용이 있는 경우 숫자로 구분
  • 비슷하게 연속되는 콘텐츠의 경우 ul 태그 내 li 태그로 작성
  • 최상위 공통 선택자 *는 되도록 사용하지 않음

css

  • 모든 페이지에 normalize.css와 fonts.css 연결
  • 가장 작은 viewport 사이즈는 width 값 375px로 설정
  • @charset “utf-8”; 선언 후 작성
  • 연결된 html 태그 작성 순서를 고려하여 순차대로 진행
  • width 등 사이즈에 되도록 소수점까지 표기하지는 않음
  • a 태그의 display 값을 block 또는 inline-block으로 설정하여 width 값, height 값 지정

script

  • jQuery 사용을 위해 jquery-3.6.0.min.js 파일 사용
  • 슬라이드 메뉴에 swiper-bundle.min.js 파일 사용
  • 제작한 js파일의 경우 head 영역에 작성, defer 값을 부여

files

  • 각 html과 연결하는 css, script는 외부문서로 연결
  • 파일명은 알파벳과 언더바로 구성
  • 같은 파일 종류대로 폴더에 정리

PROJECT GOAL

ELLE KOREA는 세계적인 프랑스 패션 미디어 elle의 한국 웹페이지입니다.

이 프로젝트는 웹과 모바일 화면의 반응형으로 제작되었으며, 화려한 비주얼과 다채로운 콘텐츠로 많은 구독자들을 보유하고 있는 엘르의 트렌디한 성격을 부각하여 구현하려는 목표를 가지고 시작하게 되었습니다.

  • 맨 밑 자동으로 흐르는 글자는 white-space 값에 nowrap을 부여하여 줄바꿈이 일어나지 않게 함. width값을 부여하여 css animate로 제어.
  • 카드부분은 jQuery로 객체를 선택하여 mouseenter, mouseleave로 animate 값을 이용하여 자연스럽게 움직임을 주고, 각 카드의 포지션 값을 조절하여 구현
  • YTPlayer를 사용하여 유튜브 영상 삽입
  • 자바스크립트를 적용하여 스크롤을 내리면 맨 위 메뉴 바의 높이가 작아지게 함

가장 오랜 시간을 걸쳐 제작한 페이지입니다. swiper.js를 사용하여 slide의 움직임이 자동으로 구현 되었을 때 신기하고 뿌듯함이 컸습니다.

특히 아랫쪽 흐르는 글자는 글자 크기가 커서 그런건지 어떤 소스를 적용해도 제어가 잘 되지 않고 뚝뚝 끊기는 현상이 일어나 수정에 수정을 거듭했습니다. 각 slide에 width 값을 부여하고 그것을 둘러싼 태그에 width값을 calc(width * slide 개수)를 사용하여 결과적으로 가장 마음에 들게 구현되어 다행이었습니다. 이 작업으로 움직이는 글자 제어는 어느정도 이해가 되었다고 말할 수 있습니다.

체감상 정말 오래 고민하고 괴로웠지만 해당 작업 후 웹 페이지의 구조를 짜는 것과 자바스크립트를 적용하는 것에 어느정도 감이 익게 되었습니다. 그런 만큼 큰 애정이 담긴 프로젝트입니다.
