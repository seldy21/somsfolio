HTML

  • w3c validator 검사
  • 주 브라우저는 chrome을 기준으로 작업
  • 자식, 자손 요소 들여쓰기. tab 한 번은 4번의 공백으로 설정
  • div 태그에는 필히 class 값 작성
  • class 값은 해당 콘텐츠와 최대한 연관된 단어로 작성. 비슷한 내용이 있는 경우 숫자로 구분
  • 비슷하게 연속되는 콘텐츠의 경우 ul 태그 내 li 태그로 작성
  • 최상위 공통 선택자 *는 되도록 사용하지 않음

css

  • 모든 페이지에 normalize.css와 fonts.css 연결
  • 가장 작은 viewport 사이즈는 width 값 375px로 설정
  • @charset “utf-8”; 선언 후 작성
  • 연결된 html 태그 작성 순서를 고려하여 순차대로 진행
  • width 등 사이즈에 되도록 소수점까지 표기하지는 않음
  • a 태그의 display 값을 block 또는 inline-block으로 설정하여 width 값, height 값 지정

script

  • jQuery 사용을 위해 jquery-3.6.0.min.js 파일 사용
  • 슬라이드 메뉴에 swiper-bundle.min.js 파일 사용
  • 제작한 js파일의 경우 head 영역에 작성, defer 값을 부여

files

  • 각 html과 연결하는 css, script는 외부문서로 연결
  • 파일명은 알파벳과 언더바로 구성
  • 같은 파일 종류대로 폴더에 정리

포트폴리오 작업 중 그나마 가장 무난하게 작업했던 프로젝트입니다. 반응형으로 제작하면서 상품 배치에 많은 고민을 했습니다.

처음에는 화면이 작아지면 상품 슬라이드를 세로로 세우고, 화면이 넓어지면 슬라이드를 가로로 배치할 수 있도록 계획했었는데 모바일로 화면을 보는 경우 슬라이드의 움직임과 스크롤이 구분되기 어려울 것 같아서 슬라이드 방향은 가로로 그대로 두고 breakpoints로 슬라이드의 수를 조절했습니다.

다만 아쉬운 점은 web에서 보이는 category의 각 섹션 높이값이 텍스트가 많은 섹션을 기준으로 일괄적으로 설정되어 다소 높게 느껴진다는 것입니다. 어떻게 설정해야 더 한눈에 편하게 보일지 조금 더 고민해봐야 할 것으로 보입니다.

간단하지만 Ajax와 JSON 파일을 적용해 본 페이지입니다. Ajax에 JSON을 연결해서 사용하는 과정에서 url을 작성하는 것이 매우 헷갈렸습니다. 이미지를 연결할 시 JSON파일을 기준으로 하는 것이 아니라 해당 페이지를 기준으로 url을 설정해야 했기 때문에 몇 번의 시행착오가 있었습니다.
